# GitHub Release Checklist

Follow these steps to publish a new release on GitHub.

## Preparation

### 1. Update Version Numbers

Update version in:
- [ ] `setup.py` (line 15)
- [ ] `main.py` docstring
- [ ] `README.md`

### 2. Test Everything

- [ ] Run `python main.py` and test full workflow
- [ ] Generate a daily report
- [ ] Test on clean Python environment
- [ ] Test setup.bat/setup.sh scripts

### 3. Build Standalone Executable (Optional)

```bash
# Install PyInstaller
pip install pyinstaller

# Build executable
pyinstaller --name="StudyFocusMonitor" --onefile --windowed main.py

# Test the executable
.\dist\StudyFocusMonitor.exe
```

## Creating the Release

### 1. Commit and Tag

```bash
git add .
git commit -m "Release v1.0.0"
git tag -a v1.0.0 -m "Version 1.0.0 - Initial release"
git push origin main
git push origin v1.0.0
```

2. Click "Releases" ‚Üí "Create a new release"
